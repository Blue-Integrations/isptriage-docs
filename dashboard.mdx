ISP Triage offers several dashboard views. Your **default landing page** is set in user preferences.

## Standard dashboard

The main dashboard summarizes operational health:

- **Installs** — Today's and upcoming planned installs
- **Tickets** — Open and recent support volume
- **Projects & events** — Active work and calendar items
- **Subscriber / revenue metrics** — Where enabled for your company

Use dashboard cards as launch points into Plan Install, Tickets, or Billing.

## Billing dashboard

**Billing → Dashboard** (or `/billing-dashboard`) focuses on AR, payments, and billing KPIs for finance staff.

## Dashboard variants

Depending on deployment, additional views may be available:

| View | Purpose |
|------|---------|
| **Dashboard 2 / 3** | Alternate layouts and chart emphasis (e.g. subscriber growth) |
| **Adjustable dashboard** | Drag-and-drop card layout where enabled |
| **Dashboard AI** | Dashboard with integrated AI panel |

<Tip>
Ask your admin which dashboard is canonical for your team, or set your preferred **login redirect** in user preferences.
</Tip>

## Status cards

Status rows typically show counts with links into the underlying module (installs, tickets, projects, events). Click a card to drill into that area.

## GIS quick access

Some dashboards link to **GIS** for map-based network views. Map center and layers are configured per company.

<Note>
Dashboard numbers refresh on page load. For real-time network status, use **Monitoring** or vendor controller pages.
</Note>
